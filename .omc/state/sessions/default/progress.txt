# Progress Tracking - Fix Monorepo Problems

## Iteration 1

### Completed Stories
- **US-001 Baseline Assessment**: Identified build failures in packages/orchestrator (tsgo missing @types/node), apps/web (vinxi/vite8 manifest path incompatibility), apps/music-scanner-service (missing .js import extensions). Identified test failures in packages/core (mock arrow functions not constructible in vitest v4), packages/adp (race conditions in WebSocket tests). Identified 1854 lint warnings (mostly in build artifacts).

- **US-002 Build System Migration (vite+)**:
  - Replaced `tsup` with `tsdown` (powered by Rolldown) in `packages/core` and `packages/adp`.
  - Replaced `tsc` with `tsgo` (TypeScript v7 beta / @typescript/native-preview) in `packages/orchestrator` and `apps/music-scanner-service`.
  - Added `@typescript/native-preview` and `tsdown` to root devDependencies.
  - Updated `pnpm.overrides.vite` to `^8.0.0` to align with vite-plus.
  - Fixed `apps/music-scanner-service` import extensions for ESM (`./tools/search.js` etc).
  - Added default export to `apps/web/app/client.tsx` to satisfy vinxi handler.
  - Patched `vinxi@0.5.11` to support Vite 8 manifest paths (`vite-plus` provides Vite 8).
  - Added `@types/node` to `packages/orchestrator` and set `"types": ["node"]` in its tsconfig for tsgo compatibility.
  - Updated `packages/core` and `packages/adp` `package.json` exports to match `tsdown` output (`.cjs`, `.mjs`, `.d.cts`, `.d.mts`).

- **US-003 Test Suite Fixes**:
  - Changed root `test` script from `vp run -r test` to `vitest run` for reliable monorepo test execution.
  - Fixed `packages/core/tests/AgentEventLoop.test.ts` mocks: changed arrow functions to regular `function()` so they can be used with `new` in vitest v4.
  - Fixed `packages/adp/tests/client.test.ts` race conditions: added dummy server handler for `Test.timeout`, prevented double `server.close()`, changed `AdpClient.close()` to use `ws.terminate()` for reliable test cleanup.
  - Added `ws.on("error", () => {})` in `AdpClient` constructor to prevent unhandled errors during test teardown.

- **US-004 Script Completeness**:
  - Verified all packages have valid `build` and `test` scripts.
  - Verified service packages have valid `start`/`dev` scripts.

- **US-005 Linting Resolution**:
  - Updated root `lint` script to `vp lint --type-aware --type-check`.
  - Created `.eslintignore` to exclude build artifacts (`apps/web/.output`, `apps/web/.vinxi`, `dist`, `scratch`).
  - Added `"types": ["node"]` to root `tsconfig.json`.
  - Fixed `no-unused-vars` across source files (removed unused imports, prefixed unused params with `_`, removed catch params).
  - Fixed `no-floating-promises` by adding `void` prefix.
  - Fixed `no-base-to-string` by casting `ws` message data to `Buffer`.
  - Fixed `restrict-template-expressions` by wrapping `unknown` values with `String()`.
  - Fixed type errors in `apps/web` (TanStack Start API changes for `StartClient` router prop and `createStartHandler` signature).
  - Fixed type errors in `apps/pi-extension` (workspace import instead of relative `.ts` path).
  - Fixed type errors in `packages/agx-cli` (declared `box` as JSX intrinsic element via `@ts-expect-error`).
  - Fixed type errors in `packages/orchestrator/tests` (added missing required fields to mock objects).
  - Fixed `apps/orchestrator-demo` types (added missing fields, removed non-existent `round` property).

- **US-006 Final E2E Verification**:
  - `pnpm run build`: 8/8 tasks pass
  - `pnpm run test`: 13/13 test files pass, 68/68 tests pass
  - `pnpm run lint`: 0 warnings, 0 errors
