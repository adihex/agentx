# web

## Purpose

`@agentx/music-extractor-web` is the browser frontend for the AgentX Music Scanner. It renders a single-page UI where a user enters a song name and watches a multi-step extraction workflow (search -> download -> extract -> done) progress in real time. The UI opens a WebSocket to the agentx runtime (`ws://localhost:9222`), sends an ADP (Agent Debug Protocol) JSON-RPC `Music.StartExtraction` request, and reacts to `Music.Status` and `Toolchain.responseReceived` messages to drive a progress bar, step indicator, and system log. The app currently carries two parallel entry points: a TanStack Start SSR app under `app/` (file-based routing, the intended production target) and a legacy plain-Vite SPA under `src/` (mounted via `index.html` -> `src/main.tsx`). Both render the same component set and the same `Music Scanner` screen.

## For AI Agents

### Working In This Directory

Scripts (from `package.json`; run from this directory):

- `npm run dev` — start the Vite dev server on port 3000.
- `npm run build` — `vite build`.
- `npm run start` — `vite start`.
- `npm run test` — `vitest run` (single pass). `npm run test:watch` for watch mode.
- `npm run test:e2e` — `playwright test` (auto-starts the dev server on port 3000).
- `npm run storybook` — Storybook dev server on port 6006. `npm run build-storybook` to build static Storybook.

Framework specifics:

- TanStack Start (`@tanstack/start`) + TanStack Router (`@tanstack/react-router`) with SSR via `app.config.ts` (`node-server` preset). React 19 (`catalog:`-pinned). `clsx` + `tailwind-merge` are present for class composition; icons come from `lucide-react`.
- Routing is file-based under `app/routes/`. Each route exports a `Route` created with `createFileRoute(path)` (or `createRootRoute` for `__root.tsx`). `app/routeTree.gen.ts` is generated from these files per `tsr.config.json` — never edit it manually; add/rename route files and let the generator update it.
- Important caveat: two entry points coexist. `index.html` loads `src/main.tsx` (legacy SPA, no SSR/routing), while `app/client.tsx` / `app/ssr.tsx` drive the TanStack Start app. The component logic is duplicated between `app/components/` and `src/components/`, and `app/routes/index.tsx` mirrors `src/App.tsx`. When changing UI or ADP behavior, update both copies (or consolidate) to avoid drift.
- Backend contract: the UI connects to `ws://localhost:9222` and speaks JSON-RPC 2.0. It sends `Music.StartExtraction` with `{ songName }`; it handles inbound `Music.Status` (log message) and `Toolchain.responseReceived` (`toolName` of `searchMusic` / `downloadAndUpload` / `triggerCloudRun` drives step + progress). On socket close it auto-reconnects after 3s.

### Testing Requirements

- Unit/component tests live in `src/components/__tests__/` and run via Vitest (jsdom + Testing Library, configured in `vitest.config.ts` and `vitest-setup.ts`). Run with `npm run test` here, or via Vitest from the monorepo root. Note `vitest.config.ts` declares 100% coverage thresholds (statements/branches/functions/lines) — coverage runs will fail below 100%.
- e2e tests live in `e2e/` (`scanner.spec.ts`) and run via Playwright across chromium, firefox, and webkit. `npm run test:e2e` starts the dev server (`npm run dev`, port 3000) automatically and reuses an existing one outside CI.
- Component assertions rely on accessibility/semantics: `ProgressBar` exposes `role="progressbar"` with `aria-valuenow/min/max`; `StepIndicator` items carry `data-testid="step-<id>"`.

### Common Patterns

- Function components typed as `React.FC<Props>` with a local `interface ...Props`; presentational components are stateless and driven entirely by props.
- All styling is inline `style={{...}}` objects (dark theme: `#0a0a0a`/`#1a1a1a` backgrounds, `#00f2fe`/`#4facfe` cyan-blue accents). No CSS modules or Tailwind classes in use yet despite `tailwind-merge` being a dependency.
- Workflow state in the screen component is a small `useState` set (`songName`, `status`, `logs`, `progress`, `activeStep`, `currentSongLabel`) plus a `useRef<WebSocket>`; `addLog` keeps only the last 5 log lines.
- A couple of `@ts-expect-error` comments in `app/client.tsx` paper over a React 19 vs. `StartClient` prop type mismatch.
- Storybook stories are colocated as `*.stories.tsx`; `.storybook/main.ts` only globs `app/components/**`, so stories must live under `app/components/` to be picked up.

## Dependencies

### Internal

None. This package declares no `@agentx/*` workspace dependencies; it integrates with the rest of AgentX at runtime over the ADP WebSocket (`ws://localhost:9222`), not via package imports.

### External

- Framework/runtime: `@tanstack/start`, `@tanstack/react-router`, `react` / `react-dom` (v19, `catalog:`), `react-router-dom` (legacy, present but unused by the TanStack app).
- UI utilities: `lucide-react` (icons), `clsx`, `tailwind-merge`.
- Build/tooling: `vite` + `@vitejs/plugin-react`, `typescript`, `tsx`.
- Testing: `vitest`, `jsdom`, `@testing-library/react`, `@testing-library/jest-dom`, `@playwright/test`.
- Storybook: `storybook`, `@storybook/react`, `@storybook/react-vite`, `@storybook/blocks`, `@storybook/test`, and addons (`addon-essentials`, `addon-interactions`, `addon-links`).
