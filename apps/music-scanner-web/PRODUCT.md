# Product

## Register

product

## Users

- Guitarists looking to isolate guitar tracks/stems for practice or analysis.
- AI developers testing the AgentX runtime and the ADP socket interface.
- General music enthusiasts wanting to split stems easily.

## Product Purpose

A browser frontend for the AgentX Music Scanner. It provides an interface to submit a song name, monitor progress through the multi-step extraction pipeline (search -> download -> extract -> complete), and inspect live system logs streamed from the agent. Success is a highly reliable, responsive, and visually cohesive utility.

## Brand Personality

Sleek, precise, and terminal-inspired. It evokes the feeling of a high-tech audio workstation or developer terminal—focused, clean, and utilitarian, but premium and state-of-the-art.

## Anti-references

- Saturated SaaS "cream/sand" backgrounds.
- Hyper-rounded corners (keep radii between 8px and 12px).
- Generically repeated card-grids and bloated illustration spacers.
- Loud gradient text and neon-rainbow border glows.

## Design Principles

1. **Tool-First Precision**: The interface should feel like a dedicated hardware device or professional terminal tool, where utility is prioritized and every pixel serves a purpose.
2. **Live Feedback Integrity**: System status, logs, and progress bars are the core value. They must render without delay, represent actual runtime events accurately, and remain fully legible.
3. **High-Tech Restraint**: Visual flair is carried by layout structure, crisp typography, and status-color accents (green/cyan for connected/connected, red/yellow for error/warning/active), not decorative gradients or blurs.

## Accessibility & Inclusion

- Clear typography and sufficient contrast (>= 4.5:1) for all system logs and text.
- Full respect for user accessibility preferences including reduced motion transitions.
- Interactive elements must be semantically clean and identifiable by assistive technologies.
